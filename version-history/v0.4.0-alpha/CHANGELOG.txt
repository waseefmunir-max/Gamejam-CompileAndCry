SKYBOUND: GODFALL — v0.4.0-alpha "Kickoff"
Changes since v0.3.0-alpha
File: skybound_godfall_v0.4.0-alpha.html
========================================================

MAJOR SYSTEMS BUILD. This is the largest single jump in the project —
nearly every core game system either debuts here or is rebuilt.
Title changes to SKYBOUND: GODFALL — KICKOFF.

ADDED — Kickoff theme integration
- "Kickoff" branding replaces "SkyLink" throughout (triggerKickoff,
  triggerReKickoff) — relay-destruction start-of-level mechanic now
  explicitly tied to the jam's "Kickoff" theme

ADDED — Wave system
- Levels now split into waves (startWave, clearWave, spawnWaveEnemy,
  waveEnemyCount, waveEnemyHp) that escalate in count and toughness
- Difficulty-based wave counts (Rookie 3 / Veteran 4 / Extreme 5)

ADDED — 10-level structure with biomes
- Full 10-level roster defined, each with name, tagline, biome
  (ocean/desert/field/valley/snow/jungle/swamp/volcanic/canyon/
  godfall-void) and boss slot assignment
- drawBiome for per-level visual identity

ADDED — Combat expansion
- Missile system (fireMissile, explodeMissile)
- Weapon power-ups: triple/spread/damage (grantWeapon, milestoneReward)
- Chain-kill milestones now grant real rewards (shield/boost/missiles/
  weapons at combo thresholds) instead of score-only payoff

ADDED — Difficulty tiers
- Rookie / Veteran / Extreme selection screen (drawDifficulty) with
  distinct scaling

ADDED — Save & settings systems
- 10 manual save slots (saveSlot, loadSlot, getSlots) plus quick save
  (saveQuick) and autosave (saveAuto)
- Settings persistence via localStorage (persistSettings, loadJSON/
  saveJSON)
- Progress persistence across sessions (persistProgress)

ADDED — Pause menu & controls
- Pause menu (drawPause) with Resume/Restart Level/Restart Wave
  (restartLevel, restartWave)/Main Menu
- Volume sliders (drawSliders, handleSliderDrag, syncVolumes)
- Control remapping screen (drawControls)

ADDED — Cheat system (first pass)
- GOD MODE toggle (settings.godMode) — grants invincibility and
  10x damage; entry screen exists (drawGodCode) but is not yet
  code-gated (see Known Issues)

ADDED — Audio pipeline
- playSfx / preloadAudioBase / musicMode / refreshMusic — structure
  for loading external audio files with procedural fallback

ADDED — Screens
- drawSaves, drawSettings, drawComplete, drawVictory, drawGameOver
  as distinct dedicated screens (previously merged/implicit)

REMOVED (superseded)
- igniteSky, spawnKickoffWave, onKey, spawnEnemy, spawnObst,
  spawnCoins, sprite, blob, mixc, drawPanel — replaced by the
  wave/spawn/rendering systems above

KNOWN ISSUES (carried into v0.5.0 fix pass)
- GOD MODE toggle is reachable and functional WITHOUT entering the
  unlock code — the code-entry screen exists but doesn't gate access
- Level 10 (GODFALL) still runs through a normal wave sequence before
  the boss instead of dropping straight into the "final encounter" —
  can produce a stuck/unclear state since LV10 was designed as
  boss-only
- Restarting a wave can leave the Kickoff relay in a state where it
  doesn't respawn, softlocking that retry path
- Pause → Controls/Save-Slots screens don't reliably return to the
  paused game afterward
- HUD elements from the old code haven't yet been reworked with
  waves===0 guards for the LV10 boss-only edge case
- Takeoff prompt still refers to the original hardcoded key name
  rather than the player's remapped key
