SKYBOUND: GODFALL — KICKOFF EDITION
Version: v0.5.0-alpha
Previous Version: v0.4.0-alpha
Release Type: Major Gameplay and Presentation Overhaul

============================================================
OVERVIEW
============================================================

SKYBOUND: GODFALL v0.5.0-alpha introduces the Kickoff Edition,
a broad redesign of the campaign's gameplay flow, boss encounters,
environments, user interface, controls, audio, and progression systems.

The KICKOFF mechanic is now integrated into every stage of combat.

Instead of destroying only one relay at the beginning of a mission,
the player must destroy a new relay before each wave. Every relay
destruction kicks off the next assault and creates a repeated gameplay
cycle built directly around the GameJam theme.

============================================================
KNOWN LIMITATIONS
============================================================

- Game balance across all levels and difficulties remains experimental.

- Boss attacks and projectile density may require additional balancing.

- Save data may become incompatible with later major versions.

- Active player bullets and missiles are not preserved when restoring a
  saved game.

- External audio currently expects MP3 files using the required naming
  structure.

- External artwork and audio are optional and may not yet be available.

- Procedural fallback assets remain temporary placeholders.

- Some menus and control-binding interactions require further testing.

- Performance may decrease during projectile-heavy boss phases.

- Browser local storage can be cleared by the user or browser, removing
  saved progress.

============================================================
TECHNICAL INFORMATION
============================================================

- Built using HTML5 Canvas.

- Written with vanilla JavaScript, HTML, and CSS.

- Uses no required external libraries or runtime dependencies.

- Runs directly inside a modern web browser.

- All gameplay logic remains contained inside a single HTML file.

- Uses browser local storage for:

  - Campaign progress
  - Settings
  - Save slots
  - Autosave data

- External visual assets are loaded from:

  assets/

- External audio assets are loaded from:

  audio/

- Procedural graphics and audio are automatically used when external
  files cannot be loaded.
