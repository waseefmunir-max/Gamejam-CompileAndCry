SKYBOUND: GODFALL — v0.5.0-alpha "Kickoff Edition"
Changes since v0.4.0-alpha
File: skybound_godfall_v0.5.0-alpha.html
========================================================

FIX & POLISH PASS on the v0.4.0 systems build, plus internal cleanup.
Title changes to SKYBOUND: GODFALL — Kickoff Edition. This is the
current build.

FIXED — all Known Issues from v0.4.0
- GOD MODE is now properly code-gated: a 6-digit unlock code must be
  entered before the toggle becomes accessible (godUnlocked flag);
  previously the toggle worked with no code at all
- Level 10 (GODFALL) now correctly skips the wave sequence and drops
  straight into the boss-only "final encounter" (world.waves===0
  handling added throughout: HUD wave readout, progress bar, death
  screen, level-complete screen)
- Restart Wave no longer softlocks — the Kickoff relay now correctly
  respawns on wave restart
- Pause → Controls and Pause → Save Slots now correctly return to the
  paused game instead of losing that context
- Takeoff prompt now names the player's actual (possibly remapped)
  throttle key instead of a hardcoded default

ADDED — Boss depth
- Multi-phase boss AI (bossPhases, bossAttack) — bosses now change
  attack patterns as their HP drops, rather than a single fixed
  pattern for the whole fight
- Per-boss art variety expanded (bossArt) across all 12 unique
  boss/difficulty combinations

ADDED — Audio/music matrix
- updateMusic / musicKeyFor select tracks based on BOTH weather and
  difficulty tier, replacing the single-axis musicMode from v0.4.0
- tone / sfx / noise — refined procedural audio fallback layer

ADDED — Death/completion detail
- Game Over screen now explicitly shows which wave (or "FINAL
  ENCOUNTER" for LV10) and which boss phase the player died on

CHANGED — Internal cleanup (no functional change to most)
- Large-scale function renaming/simplification pass for readability
  and file size (e.g. spawnKickoffRelay -> spawnRelay, triggerKickoff
  -> kickOff, saveSlot/loadSlot/getSlots -> readSlots/writeSlots,
  persistSettings/persistProgress -> saveSettings/saveProgress,
  chooseNextLink -> chooseLink, registerChainKill -> chainKill)
- Save-slot storage consolidated under a single SLOTS constant with
  readSlots()/writeSlots() helpers wrapping localStorage directly

KNOWN LIMITATIONS (open going into playtesting)
- Balance is untested at this scope: takeoff difficulty, boss HP,
  coin thresholds, and wave difficulty curve all still need real
  playtesting feedback
- Sprites/audio are still procedural fallback only — real asset
  files (assets/, audio/) have not yet been dropped in
