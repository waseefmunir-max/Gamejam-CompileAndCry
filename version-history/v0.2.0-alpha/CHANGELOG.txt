SKYBOUND: GODFALL — v0.2.0-alpha
Changes since v0.1.0-alpha
File: skybound_godfall_v0.2.0-alpha.html
========================================================

FULL GENRE PIVOT. This is a ground-up rewrite, not an iteration —
the side-view runner from v0.1.0 is replaced entirely by a top-down
arcade flight-combat format. Title changes to SKYBOUND: GODFALL.

ADDED
- Top-down camera/perspective and movement (replaces side-view runner)
- Boss encounters (drawBoss, spawnBoss) — first appearance of boss
  fights in the project
- Radar / minimap (drawRadar) showing nearby entities
- Level select screen with per-level unlock logic (unlocked(),
  coinReq()) — coins now gate progression between levels
- Enemy death handling (enemyDie) and level-completion flow
  (finishLevel)
- New sprite/text rendering helpers (sprite, txt, wrap) replacing
  the old side-view drawing helpers

REMOVED (superseded by rewrite)
- Side-view specific drawing: drawPlane, drawHills, drawTower,
  drawMine, cloud
- Single-lane obstacle/pickup spawners: spawnObstacle, spawnPower,
  spawnFuel (replaced by top-down enemy/pickup spawning)
- Old level-select screen (drawLevelSelect) and text helpers
  (text, wrapText)

CARRIED OVER
- Fuel meter, shield mechanic, weather variation, procedural audio
  (all re-implemented for the new perspective, not new concepts)

KNOWN LIMITATIONS
- No combo/chain mechanic yet — combat is still kill-for-score only
- No wave structure within levels (levels are single continuous
  encounters)
- No save system
- Single difficulty tier
